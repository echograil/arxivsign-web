ArxivSIGN · About

每天追论文，太累了

arXiv 每天发布数百篇 AI 论文。ArxivSIGN 是一个自用工具：自动采集、AI 改写成播客摘要、语音合成，每天生成一份可以听的论文日报。从想法到线上跑通，完整经历了数据、模型、前端、部署四个层面的设计与迭代。

几个主动的设计选型
格式优先于功能
Step 1
采集阶段就确定了统一的 paper_key 格式和字段结构。这个决定让后续每一步都能独立演化——Step2 不关心采集逻辑，Step4 不关心摘要生成，各步之间只通过 JSON 文件交换数据。后来 merger.py 出现新来源覆盖旧数据的问题，因为格式稳定，只改了 save_result 一个函数，其余管线不需要动。
生产和评判分离，不让同一个模型既写又审
Step 2
最初尝试让 LLM 质检员判断"有没有元评论混入正文"，发现它把"必须有误导性说明"当成错误来扣分——质检员的元认知不可靠。改成生产者专注生成、打分员只问"听众30秒内能不能抓住重点"，两个角色目标清晰，Best-of-5 选优后质量稳定提升，且有对照实验数据支撑（tts_ready_v2 vs tts_ready_RAG）。
卡片而不是聚类图表
Step 4
Step4 最初做的是关键词聚类可视化，发现论文关键词存在泛用词（如 "LLM"、"agent"）和高度独有词共存，自带去重效果但聚类边界模糊，信噪比低。改为按 topic_tag 分组的卡片展示，关键词降权为辅助信息，信息层级更清晰，也更容易接入 TTS 音频。
静态 HTML + 分离部署，源码不暴露
部署
本地管线生成静态 HTML，推送到 GitHub Pages 展示；反馈服务单独部署到 Railway。两者完全解耦：管线代码、提示词、API Key 全部留在本地，线上只有内容和接口。更新时只需推送新 HTML，不触碰服务端逻辑。
本地与云端 API 可切换，不绑定单一方案
Step 3
TTS 模块封装了统一的 run_tts() 入口，TTS_BACKEND 变量控制走本地 sovitsTTS 还是云端 API。本地 30G 模型跑在移动硬盘，上线后切换 API 只改一行配置，管线其他部分零改动。LLM 摘要同理，经过模型选型、价格与效果权衡后确定方案，接口层隔离了具体模型的细节。
每一步的关键转折
1
采集：格式决策 + 双源合并
arXiv API · HuggingFace Papers
双源数据格式不一致，早期统一了 paper_key 命名规则和字段结构，避免了后续大量字段映射工作。
新来源数据进来时会覆盖旧数据而不是合并，排查后在 merger.py 的 save_result 加入历史累积逻辑，按 paper_key 去重追加，已有字段由新数据更新。
2
摘要：从跑通到可生产质量
LangGraph · LlamaIndex · Best-of-5
模型选型和价格效果权衡花了较多时间，提示词定型和可生产质量评估经过多轮迭代。管线跑通后回来尝试 LangGraph loop，发现质检员元认知不可靠后转向生产打分分离的 Best-of-5。
提前预设了 Step3 TTS 模型的文本处理长度限制，摘要生成阶段就控制了每段字数，避免了 TTS 截断问题。
引入 LlamaIndex RAG，以历史高分稿为 few-shot 示例，通过对照实验量化验证效果。
3
TTS：成熟管线的迁移与适配
GPT-SoVITS v4 · 中英混合
基于已有的书籍 TTS 管线改造，复用了路径哈希缓存和断点续跑逻辑。
论文摘要中英混合的特点导致纯中文模型发音异常，尝试了双语参考音频和语言参数调整，确认 text_lang="zh" 即为中英混合模式。
本地/云端 API 切换设计让上线后的迁移成本降到最低。
4
前端：从聚类图表到卡片展示
信息层级 · 交互细节
原始方案是关键词聚类可视化，发现聚类效果差后转为按 topic 分组的卡片展示，关键词作为辅助信息降权显示。
卡片展开/收起与音频播放按钮绑定在同一元素上，导致点播放时触发展开。用 stopPropagation 将两个交互完全分离。
Top5 排序逻辑与分类展示逻辑在多次迭代中偶发不同步，类似的维护不同步问题在 feedback 等多处也出现过。通过保留旧版存档和版本号命名（v2、v3、v4）保持了迭代稳定性。
值得记录的三个坑
LLM 质检员元认知不可靠
用规则约束打分员后，它把"必须有误导性说明"理解成了扣分项。放弃规则化质检，改为只问听众体验，打分稳定性和区分度都提升了。
GitHub API 写入 sha 冲突
反馈写入 feedback.jsonl 时文件 sha 过期，PUT 请求返回 409。改为每次写入前先 GET 最新 sha，再带 sha 提交，冲突消失。
TTS \n\n 被原样念出来
tts_script 存的是 JSON 转义换行符，clean_for_tts 里 replace("\n\n") 没有效果。改为同时处理转义和真实换行符，并用正则清理重复标点。
